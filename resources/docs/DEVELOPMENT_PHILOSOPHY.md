# Filosofía de Desarrollo

Studio se construye con recursos limitados.

Cada hora invertida debe acercar el producto a venderse.

No buscamos perfección.

Buscamos evolución constante.

Cada Sprint debe entregar valor.

Si una función requiere semanas de trabajo y no genera valor inmediato, se divide.

Siempre construiremos el MVP primero.

Optimizar antes que reinventar.

Automatizar antes que repetir.

Documentar antes que olvidar.
